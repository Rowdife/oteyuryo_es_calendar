# 画面設計（MVP / ES締切のみ）

対象：日本（日本語）／新卒就活生向け  
前提：締切は「募集要項（posting）」単位で管理し、MVPは ES締切のみを扱う。

---

## 1) 画面一覧（MVP）

1. トップ（一覧）: `/`
2. ES締切詳細（募集要項詳細）: `/p/[postingId]`

---

## 2) トップ（一覧）`/`

### 目的
- ユーザーが「いま確認すべきES締切」を素早く見つけ、詳細へ遷移できる状態にする。

### 表示要素（MVP）
- ヘッダー
  - サービス名
  - 検索（任意 / 後で追加可）
- フィルタ（任意 / 最初は最小）
  - タグ（業界、職種カテゴリなど）※既存実装があれば流用
- 一覧（募集要項カード）
  - 企業名
  - 募集タイトル（職種／コース）
  - ES締切（日時 + 残り日数）
  - 信頼情報（最終確認日時）
  - 出典（公式）への導線（アイコン/テキスト）

### 状態（必須）
- Loading（データ取得中）
- Empty（該当なし）
- Error（取得失敗）

### 操作
- カードクリック → 詳細 `/p/[postingId]`
- 「公式」クリック → 公式URLを新規タブで開く（追跡は後で）

### 判断基準（UX）
- 一覧で “締切” と “信頼（最終確認）” が1秒で理解できる。
- 公式ソースへ最短1クリック。

---

## 3) ES締切詳細（募集要項詳細）`/p/[postingId]`

### 目的
- その募集要項のES締切情報を「公式ソースに基づくもの」として安心して確認し、次の行動（公式確認／保存）に繋げる。

### 情報設計（上から順）

#### A. ファーストビュー（最重要）
- 企業名
- 募集タイトル（職種／コース名）
- ES締切（日時 + 残り日数）
- 主要CTA
  - `公式を見る`（必須）
  - `カレンダーに追加`（ICS / 任意・後で実装可）
- 信頼ブロック（必須）
  - 出典URL（公式）
  - 最終確認日時（JST）
  - （任意）更新履歴リンク（MVPは不要でも可）

#### B. 詳細（MVPは最小）
- 対象卒年（例：26卒）※データがある場合のみ
- 職種カテゴリ/タグ（技術、営業など）※あれば
- メモ（任意 / 後で）

#### C. 回遊（任意 / 早期にあると便利）
- 同じ企業の他募集（同社内の他posting一覧）
  - 企業名見出しの下に小さく配置
  - クリックで該当posting詳細へ

### 状態（必須）
- Loading
- Not Found（postingId不正/削除）
- Error（取得失敗）
- 出典欠落（公式URLが無い）：明示バッジ + 通報導線（MVPは「表示のみ」でも可）

### 操作
- `公式を見る` → 公式URLを新規タブで開く
- 同社の他募集 → posting詳細へ遷移

### 表示ルール（信頼を守る）
- 公式URLが無い募集要項は「参考情報」扱いにし、信頼表示で明確に区別する（MVPでは原則掲載しない方針も可）。
- 締切日時は JST 明示、日付だけの場合は “23:59 JST（推定）” のような扱いを禁止し、データの粒度をそのまま表示する（例：`YYYY-MM-DD（時刻未確認）`）。
